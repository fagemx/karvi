# 02 — 垂直與水平的新理解

> 2026-03-06 研究筆記

## 傳統理解

```
垂直 SaaS = 針對特定行業（律師、行銷、醫療）
水平 SaaS = 通用工具（CRM、專案管理、收款）
```

## 新的理解

垂直不是「行業」，是**職業**。每種職業有不同的 workflow，但底層動作是共通的。

```
垂直（職業）        水平（基礎動作）
─────────         ─────────────
律師               研究 → 草擬 → 審查 → 修改
行銷               研究 → 草擬 → 審查 → 修改
工程師             研究 → 草擬 → 審查 → 修改
                        |
                   骨架一樣，差別在：
                   - 輸入（判例 vs 受眾 vs codebase）
                   - 驗收（法律正確 vs 轉換率 vs test pass）
```

## 人人養龍蝦的世界

現在人人用 Claude。不只工程師，律師、行銷、會計都在用。這改變了一切：

### 簡單任務 — 不需要 Karvi

用戶直接跟 Claude 說「幫我寫一封行銷信」。一次性的、不需要追蹤的、不需要併發的。Claude 自己就能做。

### 複雜任務 — 需要某種基礎設施

- 多步驟、有依賴（研究完才能寫、寫完才能審）
- 多 agent 併發（同時處理 10 個案子）
- 條件會變（做到一半需求改了）
- 沒有標準答案（是判斷，不是對錯）
- 需要追蹤和控制（花了多少錢、跑到哪了、要不要停）

## 包裝者是誰？

以前假設：平台包裝 workflow → 用戶按按鈕 → agent 執行。

但如果人人都會用 AI：

| 包裝者 | 模式 | 問題 |
|--------|------|------|
| 平台包裝 | Harvey（律師 SaaS）、Jasper（行銷 SaaS） | 用戶被鎖在平台定義的 workflow 裡 |
| 用戶自己包裝 | 直接跟 Claude 說 | 複雜任務管不了、沒有追蹤和控制 |
| **AI 包裝** | AI 理解意圖 → 自己編排 pipeline | 需要可靠的執行基礎設施 |

**答案是第三個：AI 自己包裝。**

用戶說意圖，AI 理解意圖，AI 用工具組裝 pipeline，基礎設施保證執行。

## 真實世界不是「做一次然後複製」

用戶說「幫我處理這個合約」：

```
看起來像 A（標準審查）
  但其實是 B（有特殊條款）
  過程中冒出 C（發現關聯問題）
  條件會變（對方改了條款）
  產出不是對錯（是取捨判斷）
```

這不是固定 pipeline 能處理的。需要：

- **意圖理解**：A 背後是 B
- **動態編排**：遇到例外就調整 step
- **適應性**：不是重試是重新理解
- **判斷力**：產出是建議 + 理由 + 替代方案

## 對 Karvi 的意義

Karvi 不是給用戶的產品，不是給特定職業的 SaaS。

**Karvi 是給 AI 用的基礎設施。** AI 是編排者，用 Karvi 的工具組裝和執行 pipeline。用戶只需要跟 AI 說意圖。
