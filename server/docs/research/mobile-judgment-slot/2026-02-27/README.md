# Mobile Judgment Slot 研究軍團（2026-02-27）

目標：定義 task-engine 的手機端本質與 v0 產品規格，避免 SaaS dashboard 路線。

## 軍團分工

1. `A-mobile-habit-model.md` — 手機使用習慣與中斷模型
2. `B-blackboard-interaction-grammar.md` — 黑板互動語法（signal → interpretation → commitment → rule）
3. `C-notification-attention-strategy.md` — 通知策略與注意力成本
4. `D-failure-modes-guardrails.md` — 失敗模式與防護機制

## 產出要求

- 聚焦「人類判斷注入（human judgment slot）」而非任務審批。
- 每份報告要包含：原則、反模式、v0 實作建議。
- 文字以繁中為主，條列清楚，可直接轉成產品規格。
