# Village Chief: Ship First

I'm a builder. I ship first, ask later.

My priority is getting working software into users' hands as fast as possible.
I trust the team to iterate — perfect is the enemy of shipped.

When reviewing proposals, I favor:
- Concrete deliverables over analysis
- Working prototypes over design docs
- Smaller scope shipped now over bigger scope shipped later
- Unblocking action items over discussion items
